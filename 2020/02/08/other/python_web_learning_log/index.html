vscode 配置python虚拟环境

发表于 2020-02-08 | 更新于: 2020-02-08
字数统计: 389 | 阅读时长 ≈ 1

#####写在前面：
使用python进行开发时，难免要使用虚拟环境来将不同的项目隔离开。这篇文章提供了三种方法供大家参考。

vscode可以自动检测工作区文件下的python虚拟环境

打开命令面板(ctrl+shiift+p),输入python：select interpreter，可以看到出现了两个python解析器(环境)，一个是虚拟环境(括号中有你的虚拟环境名字)，一个是原来的环境。选择虚拟环境就OK了。

备选方案

如果vscode没有检测出来的话，或许是因为文件目录太多(我猜的)
可以新开一个vscode窗口(ctr+shift+n)，在工作区只打开你虚拟环境所在目录，重复上面的步骤，这样就可以找到你的虚拟环境了。

备备选方案

实在不行，手动配置。
手动配置有两个方法。

  1. 打开 文件->首选项->设置，搜索python.venv
    Venv Folders是你的虚拟环境(你可以创建很多不同名字的虚拟环境，所以你看有个添加项按钮，可以添加好多个虚拟环境)；Venv Path是你的虚拟环境所在目录(只能填一个，也就意味着你的虚拟环境最好全放在一个目录下)
  2. 在python的配置文件中直接把虚拟环境的python.exe填上去 记得把上面的路径改成自己的哦。
